Adding TSP++ Support for TAPI 3.0 and 3.1

This version of TSP++ includes support for TAPI 3.0 for Windows 2000 or Windows XP.

In order to use the TAPI 3 support, you will need the latest version of TAPI.H and TSPI.H.  Both of these may be obtained from the Windows 2000 Platform SDK.  If the following definitions are not in TSPI.H then you need a newer version of the headers:

TSPI_lineGetCallIDs
TSPI_lineMSPIdentify
TSPI_lineCreateMSPInstance
TSPI_lineCloseMSPInstance

By default, the TAPI 3 support is not compiled when you build the TSP++ library. This is to allow the library to be used with older versions of TSPI.H which don't have the newer definitions. To enable TAPI 3 support, you will need to recompile the source code into new libraries with the TSPBuild batch file using the command-line option "tapi3".  For example:

C:\TSPLIB3> TSPBuild tapi3
Building TAPI 3.x compatible libraries
Microsoft (R) Program Maintenance Utility Version 6.00.8168.0
Copyright (C) Microsoft Corp 1988-1998. All rights reserved.
....

The derived TSP project must then correctly set the TAPI_CURRENT_VERSION flag to 3.0 or better to add the proper TAPI support into your project. This is critical to ensure that the TAPI structures and functions match up with your compiled version of TSP++.  If you selected TAPI 3.x as your target in the TSPWizard, this step is done for you.  To verify this look in the stdafx.h file for the following line:

#define TAPI_CURRENT_VERSION 0x00030000

This line must be present before tapi.h or tspi.h are included in the project.
